# Connectors -- Training Plan Plugin

This plugin is self-contained and works without external connections.

## Optional Integrations

| System | Purpose | Notes |
|--------|---------|-------|
| File system | Read/write training plans, transcripts, checkpoints, learner profiles, concept files | Required for all operations |

## Currently Active

- **File system**: Training plans, session transcripts, checkpoint files, learner profiles, and concept files are saved to the current working directory as Markdown

## Concept-Builder Integration (Fully Integrated)

As of v0.3.0, the concept-builder methodology is fully integrated into training-plan. The training-plan plugin's `concept-explainer` skill contains the complete concept-builder methodology (6-section structure, CTQ derivation and failure modes, concept maps, comparison protocol, deep-dive protocol) adapted for TTT session context.

There is no runtime dependency on the concept-builder plugin. If concept-builder is also installed, concept files generated by training-plan are structurally compatible with concept-builder's `/library` command due to shared file format and YAML frontmatter conventions.

### What Was Integrated

| Concept-Builder Skill | Integrated Into |
|----------------------|-----------------|
| concept-builder-methodology (6-section structure, depth levels, LaTeX) | concept-explainer skill |
| concept-maps (node types, relationship labels, depth scaling) | concept-explainer skill |
| ctq-methodology (CTQ derivation, failure mode taxonomy) | concept-explainer skill |

| Concept-Builder Command | Training-Plan Equivalent |
|------------------------|-------------------------|
| `/explain` | Automatic during Phase 3 TEACH (full concept explanation for complex gaps) |
| `/deep-dive` | In-session deep-dive on any taught principle (mention once after first block) |
| `/compare` | Automatic for conflation gaps (concept comparison format) |
| `/map-only` | Phase 3.5 concept map (standard depth, 10-15 nodes) |
| `/library` | `/concept-library` command (list, search, path, link) |

### Concept Files

Concept files generated during TTT sessions use the naming convention `concept-<domain>-<topic>.md` with YAML frontmatter field `type: ttt-session`. This distinguishes them from standalone `/explain` files while maintaining full format compatibility.
